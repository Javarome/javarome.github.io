I write software for more than three decades, but I started earlier.

I've worked as a developer (first Java application server from scratch at IBM), consultant & trainer (Valtech, including writing courses) and tech lead of editors' (Libelis/Xcalia, Xooloo, Beam) or service providers' (Traveldoo, Zelros, Famicity, Arianee) teams.

My expertise spans from front to back ends. On the front end, I like CSS challenges, and care for RWD, accessibility and good design practices.

On both ends, I care about:

  • Good design. I'm part of the OOP team, not the functional one. Most of my time is not spent searching how to do things, but what is the best (flexible, evolutible, maintainable, testable, scalable, performant) way to do them.
  • Minimal dependency, being as vanilla as reasonably possible: avoiding frameworks and libraries as much as possible. Don't call me to build your next React app. I prefer to take code from a library and customize it to my needs than blindnessly adding a whole bunch of general-purpose code as a dependency.
  • Using my brain on each issue instead of just following trends. I like creativity, thinking out of the box, always challenging commonly accepted ideas to make sure they make sense.

Finally, I found myself working a lot better remotely.
